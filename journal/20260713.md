# 2026-07-13 — Firmware & Gateway Preprocessing Build

## Summary

Built the passive RSSI+CSI sniffer firmware in two forms (ESP-IDF v5.4 and a
single-sketch Arduino IDE port) and the Python `gateway/` preprocessing
package that turns raw captures into synchronized per-window RSSI + D-CFR
feature records. Everything is unit-tested (44 tests) and verified
end-to-end on synthetic data. No hardware tested yet at this stage.

## Log

- Reviewed `sections/methodology.tex` to extract requirements: per-packet
  tuple `(id, t, RSSI, CSI, anchor)`, 4 anchors A1–A4, passive sniffing of a
  hidden non-cooperative device, gateway performs wanted/unwanted
  classification, D-CFR computed gateway-side (not on the ESP32).
- Locked design decisions: ESP-IDF v5.4 native APIs; **one combined firmware**
  where the CSI callback alone yields the whole tuple (`wifi_csi_info_t`
  carries MAC + RSSI + timestamp + CSI for the same packet); USB serial CSV at
  921600 baud primary, optional WiFi UDP behind a flag; **no on-device MAC
  filtering** (gateway classifies).
- Built `firmware/anchor/` (ESP-IDF): `sniffer.c` (promiscuous + CSI setup and
  both callbacks), `csv_out.c` (FreeRTOS queue + writer task, CSV/STAT
  formatting), `main.c` (WiFi bring-up ordering), `Kconfig.projbuild`,
  `wifi_conn.c`/`udp_out.c` (optional UDP mode).
- Added `firmware/tools/gateway_logger.py` — multi-port pyserial logger that
  prepends host timestamps and writes one CSV per anchor.
- Ported the firmware to Arduino IDE: `firmware/anchor_arduino/anchor_arduino.ino`
  — a single self-contained sketch with a `#define` config block, emitting the
  identical line format so the logger and pipeline work with either firmware.
- Built the `gateway/` Python package: `parser` (pure line parser tolerant of
  malformed input), `devices` (authorized-device list / tagging), `windowing`
  (host-time capture windows + 3-of-4 sufficiency), `rssi` (EWMA smoothing),
  `csi` (subcarrier remap → normalize → D-CFR), `features`, `io` (JSONL +
  summary), `preprocess` (CLI), and `synth` (synthetic session generator).
- Wrote 44 pytest tests and verified end-to-end on a synthetic session (with an
  injected anchor dropout and 2% malformed lines): malformed counts,
  usable-window rate, RSSI means, and D-CFR correlation all matched ground
  truth.
- Documented the exact line format, subcarrier mapping, and hardware gotchas in
  `firmware/README.md`.

## Issues & Fixes

- **Preprocessor `#if` inside a log/printf argument list wouldn't compile**
  (both the ESP-IDF `ESP_LOGW` banner and the Arduino `Serial.printf` banner).
  Fixed by computing the channel value into a plain variable first, then
  passing it to the single format call.

## Setup / Reproduce

- ESP-IDF build: `cd firmware/anchor && idf.py set-target esp32 && idf.py build`
  (ESP-IDF v5.4 required; `CONFIG_ESP_WIFI_CSI_ENABLED=y` is set in
  `sdkconfig.defaults`).
- Arduino build: open `firmware/anchor_arduino/anchor_arduino.ino`, board
  "ESP32 Dev Module", arduino-esp32 core 3.x, upload speed 921600.
- Python: `python -m venv .venv && .venv/bin/pip install -e '.[dev]'`, then
  `.venv/bin/python -m pytest -q`.
